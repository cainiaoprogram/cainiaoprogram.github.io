3rd-party Gradle plug-ins may be the cause

kotlin的build.gradle中添加 apply plugin: ‘kotlin-kapt时报错

Folder D:\AndroidProjects\app\build\generated\source\kaptKotlin\debug
Folder D:\AndroidProjects\app\build\generated\source\kaptKotlin\release
3rd-party Gradle plug-ins may be the cause

经过多次试验总结出解决办法
首先对照版本号，本文写作日期2018-8-5,可用的版本号为

1. Android studio version  3.1.2
2. gradle plugin version  3.1.2
3. Kotlin version  1.2.30
3. gradle wrapper version 4.4 
4. compileSdkVersion 27
5. support:appcompat-v7:27.1.1
其中android studio版本可以在Help->about中查看，官网下载相应版本
gradle version  ，Kotlin version  可以在build.gradle中更改
gradle wrapper在gradle-wrapper.properties中更改，改为
distributionUrl=https\://services.gradle.org/distributions/gradle-4.4-all.zip

记得在build.gradle（app）文件末尾添加
kapt {
generateStubs = true
}

这步走完依旧报错，呢就再走一步
在Run->Edit Cofigurations中删除instant Appprovision
