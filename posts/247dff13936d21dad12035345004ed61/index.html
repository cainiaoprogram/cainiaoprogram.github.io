最近与第三方对接时，遇到上述问题。

1.找到问题，2.解决问题

原因是第三方提供的jar包中的依赖包与我方项目中某个jar包依赖版本不一致，导致的jar冲突。

刚开始找问题没有找正确一直以为是spring版本的问题，因为项目中的boot版本用的是快照版本（建议大家使用稳定版，因为踩过坑，发布某个项目时由于jar包9点进行过更新，十点发布项目时导致发布不成功。说多了都是泪）

java.lang.SecurityException: class "org.bouncycastle.jcajce.provider.symmetric.IDEA$Mappings"'s signer information does not match signer information of other classes in the same package

主要问题还是这句话，因为这个org.bouncycastle.jcajce.provider包版本冲突，第三方jar包中引用的时4.9，而本地项目中的依赖时6.2版本导致的异常。

解决方法：先找到这个类是哪个jar包中的bcpkix-jdk15on

找到这个包的依赖，先exclude，再导入与第三方jar包依赖关系版本的jar
