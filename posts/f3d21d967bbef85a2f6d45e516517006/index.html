解决MySQL错误reading initial communication packet_MySQL的两种方法

 

打开MYSQL的时候提示reading initial communication packet，
而且网站访问出现如题reading initial communication packet（2013）错误，
处理方法一：
查找MYSQL配置文件：
检查my.cnf，发现innodb_buffer_pool_size = 18M 设置过小，调整为innodb_buffer_pool_size = 1024M即可，网上介绍是过大处理其他解决方法，解决我的问题。

Lost connection to MySQL server at 'reading initial communication packet' 错误解决

如果还不能处理问题则采用下面方法;

1、修改mysql配置文件


vi /etc/my.cnf


[mysqld]段加skip-name-resolve


在这个之前要把mysql的远程访问权限打开，或者再加skip-grant-table（不推荐）


2、修改hosts.allow


vi /etc/hosts.allow


加mysqld : ALL : ALLOW
mysqld-max : ALL :ALLOW

这一次搞定！
