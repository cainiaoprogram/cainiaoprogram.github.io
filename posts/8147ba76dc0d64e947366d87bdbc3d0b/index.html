一：
1.连接的时候提示密码不对（第一天连接上了啊，我确定都是正确的，怎么过了一夜就连接不上，怎的你俩闹矛盾了）
解决：打开sql plus

（我是用sqlplus 和navicat都用户密码错误）…
输入：sys/xxxxxxx as sysdba (xxxxxxx是你上面安装的时候设置的密码)
2. 解除锁定账号（例如解除system用户）
alter user system account unlock;
3.为system用户设置新密码（例如设置密码为(推存设置)：manager）
alter user system identified by manager;
ok(o谢特)

导入数据

创建表空间
CREATE TABLESPACE pty(表空间的名字)
DATAFILE ‘E:\Oracle\pty.dbf’ (这边可以写成oracle的某个路径下)
SIZE 50M（初始大小）
AUTOEXTEND ON;（自动扩展）

创建用户
CREATE USER xxx（用户名）
IDENTIFIED BY 123456（密码）
DEFAULT TABLESPACE pty(上面创建的表空间)

赋权限
grant dba to xxx;

–新建包结构，放置dmp文件
create or replace directory datadir as ‘E:/Oracle’;

赋权限
grant read,write on directory datadir to xxx;

上面的打开sqlplus窗口完成

下面的win+r 打开完成

impdp xxx/xxxx dumpfile=aaa.dmp logfile=boss.log full=y remap_schema=boss:xxx directory=datadir
