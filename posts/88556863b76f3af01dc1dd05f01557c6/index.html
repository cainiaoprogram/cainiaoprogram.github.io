Git将远程强制同步到本地

有时候不小心将本地代码修改后，会有将远端的代码强制同步到本地的需求，这个时候有很多人会选择删除之后重新将远程仓库的内容clone下来，其实有更好的办法来解决这种问题。

正确的做法应该是：

git fetch --all

git reset --hard origin/master

git fetch 只下载远程仓库的内容，不做任何的合并，git reset 把HEAD指向刚刚下载的最新版本。
