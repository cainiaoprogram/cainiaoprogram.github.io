5G学习笔记之NG-RAN系统架构（1）

参考资料：38.401

NG-RAN架构的一般性原则如下：
1. 信令和数据传输网络逻辑分离；
2. NG-RAN和5GC与传输相互独立。NG-RAN和5GC的寻址方案与传输的寻址方案没有相关性，即同一个设备中，传输功能不是NG-RAN或者5GC的一部分。
3. RRC连接的移动性完全由NG-RAN控制。

NG-RAN接口的一般性原则如下：
1. The functional division across the interfaces have as few options as possible.
2. Interfaces are based on a logical model of the entity controlled through this interface.
3. One physical network element can implement multiple logical nodes.

1. NG-RAN架构

  • NG-RAN由多个与5GC连接的gNB组成。gNB之间由Xn连接，gNB与5GC之间由NG连接。
  • 一个gNB可包含一个gNB-CU和一个或多个gNB-DU。
    ① gNB-CU与gNB-DU之间通过F1接口连接；
    ② 一个gNB-DU只能与一个gNB-CU连接。
    ③ 一个gNB-CU可以与多个gNB-DU连接。
  • gNB可支持FDD，TDD，或双工模式。
  • 承载NR PDCP用户面：
    ① 对于NG-RAN，是gNB-CU， gNB-CU-UP；对于EN-DC， MeNB，SgNB。
    ② 监视UE不活动情况，并将其上报给核心网（over E1或X2）。
  • 承载NR RLC用户面（gNB-DU）：监视UE不活动情况，并将其上报给控制面（即gNB-CU or gNB-CU-CP）。
  • UL PDCP配置由X2-C（EN-DC），Xn-C（NG-RAN），F1-C指示；DL/UL无线链路中断和恢复由X2-U（EN-DC），Xn-U（NG-RAN），F1-U指示。
     

2. gNB-CU-CP和gNB-CU-UP分离架构

  • 一个gNB由一个gNB-CU-CP，多个gNB-CU-UP和多个gNB-DU组成。
  • gNB-DU与gNB-CU-CP、gNB-CU-UP分别通过F1-C接口、F1-U接口连接；gNB-CU-CP与gNB-CU-UP之间通过E1接口连接。
  • 一个gNB-DU只与一个gNB-CU-CP连接；一个gNB-CU-UP只与一个gNB-CU-CP连接。
    注意：为了网络的恢复性，一个gNB-DU或 gNB-CU-UP可与多个 gNB-CU-CP连接。这样做的好处是：当某个 gNB-CU-CP挂了，可以切换到其它的 gNB-CU-CP，网络可继续工作。
  • 一个gNB-DU可与在同一个gNB-CU-CP控制下的多个gNB-CU-UP连接；一个gNB-CU-UP可与在同一个 gNB-CU-CP 控制下的 多个DU 连接。
    注意：
    ① gNB-CU-CP利用承载上下文管理来建立gNB-CU-UP与gNB-DU之间的连接。
    ② gNB-CU-CP为UE选择合适的gNB-CU-UP。
  • Data forwarding between gNB-CU-UPs during intra-gNB-CU-CP handover within a gNB may be supported by Xn-U

4. UE associations in NG-RAN Node

（1）NG-RAN node UE context：UE在NG-RAN节点中的上下文信息，用于维持活动UE的NG-RAN服务。

  • 上下文建立时机：① UE处于RRC CONNECTED态时；②切换准备期间，目标NG-RAN完成了切换资源配备。
  • 上下文至少包含的信息：① UE状态信息；② 安全信息；③ UE能力信息；④ UE相关的逻辑NG连接标识。
  • 对于双连接，在完成S-NG-RAN节点添加后，在S-NG-RAN节点中也需要建立NG-RAN node UE context。

（2）Bearer context：UE在gNB-CU-UP节点中的上下文，用于维持UE的用户面服务。其包含的信息：DRB，PDU会话，QoS流等。
