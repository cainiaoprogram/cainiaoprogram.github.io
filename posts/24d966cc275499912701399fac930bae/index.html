（3）解决办法：

最下面的java.sql.SQLException: No suitable driver才是重点

解决办法是将mysql的驱动jar包，放到D:\Program Files\Java\jdk1.8.0_91\jre\lib\ext目录下（JAVA_HOME目录下的jre\lib\ext目录下)

localhost:ext a6$ echo $JAVA_HOME
/Library/Java/JavaVirtualMachines/jdk1.8.0_131.jdk/Contents/Home
localhost:ext a6$ cd /Library/Java/JavaVirtualMachines/jdk1.8.0_131.jdk/Contents/Home
localhost:ext a6$ cd /jre/lib/ext

将下载的MySQL驱动jar包，放到jre\lib\ext目录下面

 sudo cp mysql-connector-java-5.1.44-bin.jar /Library/Java/JavaVirtualMachines/jdk1.8.0_131.jdk/Contents/Home/jre/lib/ext

适合我电脑的MySQL驱动jar包版本是mysql-connector-java-5.1.44，下载地址如下：

链接: https://pan.baidu.com/s/12dpXWNnmqZiizNV_vDp13g 密码: hy9m

然后就解决了这个问题

（4）错误场景还原需要

首先，需要在本地创建数据库spark_db，同时创建test_person数据，具体如下：

create database spark_db;

CREATE TABLE `test_person` (
  `id` int(10) NOT NULL AUTO_INCREMENT,
  `name` varchar(100) DEFAULT NULL,
  `age` int(3) DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB AUTO_INCREMENT=2 DEFAULT CHARSET=utf8;
