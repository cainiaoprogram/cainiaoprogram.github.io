使用 git 遇到 Merge branch 'master' of 的问题解决

我是没有文件冲突的情况下

  • Merge branch ‘master’ of 是在版本冲突的情况下发生的 在你 push 的时候已经有人先 push 了，导致线上的版本高于本地版本 发生的冲突
  • TortoiseGit -> Rebase…
  • 选中 force rebase 可以解决这个问题
  • start rebase 后 ，log 里的 Merge branch ‘master of’ 消失了，然后 pull ，再 push 。
  • 原因 可能是header 头指向的不一样导致冲突了，
  • 希望这篇文章能帮到你
