菜鸟程序员博客

国产实时操作系统+intel x86/龙芯平台超边缘计算机方案

引言

近年来，物联网、云计算、机器学习和网络安全等技术不断推动工业 4.0 的发展，“云边端” 的架构正逐步替代 “云管端”，边缘计算成为新时代许多领域转型的关键要素。以智能制造为例，不同于为互联网服务的 CDN 边缘计算技术，工业现场也需要性能更强、更智能的超边缘计算机，直接服务于数据源头：它能够在封闭的局域网环境中直接为 OT 设备提供智能计算服务，通过不同规格的配置和多种可扩展的硬件接口，充分满足各种工业计算场景中的算力和数据存储等需求；通过本地高速以太网或 5G 专网满足工业环境低时延和高带宽、高可靠性的网络通信需求；通过运行在超边缘计算机上的工业应用和 AI 算法模型，在本地网络环境内完成绝大多数的业务计算，大幅提升业务响应速度、降低互联网流量和带宽需求、减小数据泄露风险，从而提高企业的生产效率，同时降低总体生产运营成本。因此，翼辉信息 SuperSpirit 1 应运而生，通过强劲的性能和领先的智能边缘计算操作系统，推动企业快速数字化转型和智能制造广泛落地。

柔性工业 4.0

智能制造不仅需要高性能的计算硬件，更需要完整的软件解决方案。由信息管理层、超边缘计算层、工业控制层，上中下三层构成：信息管理层覆盖企业 IT 网络和公共互联网，企业管理员即使出差在外，也可以通过爱智云的加密连接访问企业内部的信息管理平台；在企业内部 IT 网络中，生产管理员可以通过超边缘计算机集群上的管理平台或者工业应用，查看生产状态甚至调整产线配方和生产节拍；工业控制层的自动控制器将协调产线设备并采集实时产线数据，并通过超边缘计算机上的工业应用动态下发生产配方和执行策略。

工业超边缘计算机作为超边缘计算的硬件载体，在整个方案中起到了承上启下的作用。向上，超边缘计算机可以对接运行于同样硬件平台的基于 EdgerOS 的柔性管理平台，或者功能更为强大丰富的企业管理平台。向下，运行于超边缘计算机上的 EdgerOS 工业应用通过系统服务和 API 轻松对接翼辉工业控制器 IGC1500，并由 IGC1500 完成工业 OT 侧设备的控制和数据集成。通过 MQTT、CoAP、SDDC、RTSP、ONVIF 等开放协议和 REST API，EdgerOS 应用也可以直接与工业设备通讯或者集成第三方的 IT 系统如 ERP 等。

用户针对自身业务研发的 EdgerOS 工业应用和 AI 模型，通过柔性或企业管理平台，即可在企业内实现应用的分发和部署。超边缘计算机支持 Type 1 虚拟化，可以同时运行 EdgerOS 和伴生 Linux 算力加速系统。超边缘计算层的设备接收来自管理平台的控制和基于 Web 技术的应用分发请求，同时伴生 Linux 系统也支持容器化的 AI 模型部署分发。借助 Linux 通用 AI 引擎的模型研发优势，EdgerOS 应用能够快速构建基于深度学习的机器视觉能力，实现基于超边缘计算和人工智能的柔性制造。

SuperSpirit 1 概述

SuperSpirit 1 是翼辉信息面向边缘计算推出的超边缘计算机，它有着工业级的品质、功能强大的边缘操作系统支持、强劲的性能等特点，可广泛应用于智能制造、智慧园区、智慧交通等领域。 作为边缘侧的大脑，有着远超物联网节点的算力和通信能力，其对内通过物联网技术连接工厂车间的各种传感器，实时控制各种执行机构和控制器，并应用人工智能技术实现智能化柔性制造，对外则能够通过安全连接技术连接云平台实现云计算和大数据分析。

技术特点

自主可控

SuperSpirit 1 内核为翼辉信息 100% 自主知识产权的 SylixOS。经过多年的锤炼，SylixOS 已支持几乎所有的 CPU 技术架构，并广泛应用于电力、轨交、航空航天等重要行业领域。EdgerOS 由翼辉信息结合 SylixOS 的特点和优势自主研发而成，二者相辅相成、密不可分，共同为上层应用提供了稳定、强大的 JavaScript 编程环境和各种方便好用的系统服务及 API。从此，工业应用的开发也可以使用互联网技术栈，而无须考虑繁多的通信协议和复杂的系统编程技术，应用的开发难度大大降低，Web 技术也能够帮助开发人员构建更加精致的动态用户界面。

性能强劲

SuperSpirit 1 支持 Intel® 九代Core™ i5/7 处理器，内存最高可达 32GB，并兼容 ACHI 或 NVMe 接口的高性能固态硬盘。同时它还能够通过 PCIe 接口扩展 AI 加速卡，支持 Intel® Movidius™ Myriad™ X VPU 视觉处理器芯片，算力最高可达 32 TFLOPS。

除了 SuperSpirit 1 计算节点本身的物理可扩展性，超边缘计算机集群同时支持水平方便的算力扩展能力。随着业务规模的扩大，仅需要增加更多的 SuperSpirit 1 计算节点覆盖对应的 OT 侧设备即可，所有的超边缘计算机均可通过管理平台统一协调管控。

算力共享

SuperSpirit 1 支持 Type 1 硬件虚拟化技术，能够同时运行 EdgerOS 和 伴生 Linux 算力加速系统，从而实现在同一台物理机器上同时部署 EdgerOS 应用和 Linux Docker 容器应用。无论是 EdgerOS 应用还是容器应用或模型，均能够为 OT 设备提供有力的智能计算支撑，将 OT 设备从复杂多变的业务计算中解放出来，从而专注于实时性、稳定性等专业技术指标。

例如，容器应用可以使用通用的 AI 算法框架和推理引擎（如 OpenVINO、CUDA）处理工业相机的机器视觉信号；EdgerOS 应用使用 JavaScript 则更容易编写灵活复杂的业务应用。

对于小微计算场景，使用一台超边缘计算机即可满足工业生产的业务应用和 AI 加速需求，而多台超边缘计算机之间能够互为备份，为南向的多台 OT 设备分享算力，避免传统的双机热备模型中需要多套设备冗余的成本浪费问题。

多终端和远程访问

EdgerOS 原生支持本地和远程多终端显示，客户端能够自适应多种不同的屏幕分辨率和方向，为用户提供更佳的使用体验。通过爱智云和远程终端，用户可以随时随地通过加密连接访问企业内网的管理平台，并根据需要进行远程控制。EdgerOS 支持多平台终端（Android, iOS, Windows, macOS，Linux）、多显示终端、多用户的同时访问，在不安装终端应用的情况下，通过浏览器也可以直接访问使用 SuperSpirit 1 超边缘计算机。

安全和隐私保护

作为面向边缘计算的智能操作系统，EdgerOS 充分考虑了各种计算场景的数据安全需求，能够为您的物联网设备和隐私数据提供更好的安全保护。例如，EdgerOS 的应用沙盒机制可以有效隔离第三方应用对系统数据和其它应用的破坏；EdgerOS 内置数字安全证书，即使在局域网环境中仍然对访问连接进行加密处理；EdgerOS 支持设备权限、系统服务权限、应用程序权限、用户分组权限的多维矩阵式权限模型，能够精确控制系统用户及第三方应用的各方面权限。

由于边缘计算自身的特点，SuperSpirit 1 能够在业务网络环境中完成绝大多数的计算请求从而避免数据直接上云，有效降低数据泄露风险。

强大的连接控制能力

SuperSpirit 1 配置了六个千兆以太网接口，可用于连接翼辉 IGC 工业通用控制器和工业相机。IGC 工业通用控制器是翼辉面向工业场景离散控制推出的工业控制器，可作为工业机器人、CNC 数控机床、软件 PLC 等的控制器。

IGC 工业通用控制器基于 ARM 架构处理器，运行 SylixOS 嵌入式硬实时操作系统， 支持 CODESYS、AnyControl、EtherCAT、CANOpen、ModBus 等常用工业通讯协议。翼辉信息为 IGC 工业通用控制器提供了完善的二次开发环境，支持标准的 C、C++ 语言开发，支持 POSIX 接口规范，任务周期在 1ms 以内，控制精度达到 ±10μs 以内。

SuperSpirit 1 与 IGC 工业通用控制器通过 SDDC 协议通信。SDDC（Smart Device Discovery & Control，智能设备发现与控制）是 EdgerOS 专门为 Wi-Fi 和以太网通信技术的物联网设备定义的发现与控制通信协议。使用 SDDC 协议开发的物联网设备（如 IGC 工业通用控制器）能被 EdgerOS 自动管理。

企业管理平台

除了适用于小微企业的柔性管理平台，超边缘计算机也能够接入翼辉信息 ECMP 企业管理平台。ECMP 企业管理平台为大中型企业提供完整的企业信息和组织管理能力，支撑企业构建完整的 CI/CD（持续集成、持续交付/部署）和 DevSecOps 能力。

其中，ECS Manager 是翼辉推出的边缘侧容器管理平台，它同时支持翼辉信息 ECS 轻量级安全容器、Linux Docker 容器以及 EdgerOS 应用的编排和管理。ECSM 能够添加 SuperSpirit 1 计算节点，并在边缘侧提供容器镜像管理、容器编排、容器及其宿主节点状态监控和健康管理等弹性管理能力。ECSM 使得  SuperSpirit 1 超边缘计算机具备了在边缘侧部署、升级、回滚容器和应用的功能，满足工业计算中灾备、升级等实际需求。

应用领域

智能制造

场景痛点

基于工控机 / PLC / 采集网关的产线在边缘端数据处理能力有限，工业场景下海量时序数据需要传送到云端处理，不适应工业实时要求；现场处理工业视觉往往需要专门定制固定算法的一体化硬件，造价高昂，不易调整，且不支持产线级智能化共享。

解决方案

  • 利用超边缘计算的超低延迟和边缘智能特性，集成常用工业 App，完成产线数据采集，设备综合能效管理，设备预测性维护以及故障的快速定位与处理，大幅提高了工厂的生产效率。
  • 超边缘计算平台本地分流，实现本地实时控制与调度，数据处理与加密，匹配低延时高可靠工业场景，保障核心数据安全。

  • 乱序抓取场景：针对不同的物体可以智能抓取，物品大小和形状可随意。
  • 分拣场景：把产线各种物品按照规则进行智能分拣。
  • 点胶场景：可点胶不规则的产品，比如鞋子等。
  • 拧螺丝应用场景：可智能找到螺孔，完成拧螺丝的动作。
  • 焊接应用场景：根据焊接的声音，判断焊接的质量。
  • 质量检测：智能检测产品质量和缺陷。
  • 移动作业车：辅助生产，完成智能生产。

应用效果

  • 生产过程透明可见，生产信息和在制品数据一目了然。应用 App 实时显示设备状态、传感器时域信息、设备维护提示，告警事件通过手机短信和邮件及时推送给维保管理人员。
  • 集成工业机理 + AI 算法，智能化程度高，无需人工分析，技术成本低；适用于多种设备，不需针对每台设备训练算法，降低部署时间。
  • 系统具有很强的扩展性和开放性，能通过预留标准接口实现现场设备与 MES 等系统互通。

智慧园区

场景痛点

传统 “端网云” 架构的智慧园区，智能处理和应用加载都在距离园区 “很远” 的云端，不仅数据安全和业务灵活是问题，对于低延迟和高带宽的业务也带来很大挑战；园区的特点是系统和设备的生产厂商不尽相同, 只满足了短期的某个需求, 系统之间没有相互关联。不同设备不同系统不断堆砌, 管理繁杂, 缺少统一的智能化管理平台，形成了信息孤岛。

解决方案

  • 新型智慧园区采用 “云边端” 架构，引入超边缘计算，统一融合平台，底层屏蔽多种设备接口差异。
  • 实时监测园区内测点的 AQl 指标（5 等），总和分支的水、电、热能详情，并进行趋势分析和预测。
  • 基于视频快速分析响应人员和车辆的行为识别，防疫体温红外检测体温，危险自动告警等。

应用效果

引入高体验业务，让智慧园区成为真正的智能体，升级为新一代具有网络化，数字化，智能化的有机体。

智慧楼宇

场景痛点

传统智能楼宇中采用的各种系统为封闭式系统，各系统彼此独立，不能彼此联动无法支持智能决策；各系统独立维护，维护成本高；楼宇间数据无分享，无法支撑大数据分析。因此，如何实现不同系统、不同设备之间的互联互通； 保障封闭系统走向开放后的数据安全性； 提高跨平台、跨区域的数据完整性及可靠性，是目前智能楼宇急需解决的问题。

解决方案

利用超级边缘计算机强大的人工智能和视频分析能力，配合大数据、物联网，云边端协同以及远程算法和应用部署，构建更智能化的门禁管理系统、电梯监测系统、消防监测系统、停车系统、视频监控系统、能源监控系统、照明控制系统，实现人脸识别、陌生人报警、入侵检测、位置跟踪、车牌识别、违章停车报警等智能化应用。

应用效果

  • 传统楼宇系统向智慧楼宇进化，更智能、节能、高效、 环保、安全。
  • 超级边缘计算机带来的弹性扩展，通过 MQTT 等方式对外开放共享其核心能力，为智慧楼宇后续持续进化打下基础。

总结与展望

超边缘计算机的出现为新时代的边缘计算场景提供了强有力的支撑，基于动态 JavaScript 语言的 EdgerOS 开发更简单，基于安全容器的本地应用或 AI 模型部署也更容易。随着翼辉信息柔性工业 4.0 解决方案的逐步落地，翼辉信息也将不断迭代升级计算性能更强、软件功能更智能的新型号 SuperSpirit。做为超边缘计算的核心载体，其迭代更新也将逐渐为智能制造、智慧园区、智慧交通等领域带来更多的创新机会和创造能力。标准化、自动化的云原生软件部署模型也必将推动行业转型的脚步，为中国智造的华丽转身再添绵薄一缕。
