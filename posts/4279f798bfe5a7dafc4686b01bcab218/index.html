dlp2000 开发入门

树莓派驱动 DLP® LightCrafter Display 2000 评估板

树莓派主要有驱动dpi 和i2c 两个
dpi 主要用在显示器上面的接口
mipi 的dpi

i2c 是主要控制dlpc2607芯片

dlp 的开发板p2 —> 19 20是i2c 接口
i2c 的控制方式有查看 it 的dlpc2607手册
手册地址：http://www.ti.com.cn/cn/lit/ds/symlink/dlpc2607.pdf?&ts=1589200138628

树莓派 接线图
树莓派 可以供电给dlp2000开发板，也可以外接电源， dlp2000 这个板子供有电流有大些，最好是5V 3A的电源，
树莓派供电给 dlp2000
树莓派引脚 的p2_6 :5v 引脚 接在dlp2000板上面的j3
蓝色的引脚 接的是地，另一个引脚 是+5V
电源这部有注意一下，不有接错，容易 烧板子
蓝色线接的是ged 也就是负，外接电源不能同时树莓派供电和外接5V电源，只有一个电源，

树莓派驱动dpi 的方式是在文件修的

文件在/boot/config.txt

Add support for software i2c on gpio pins

dtoverlay=i2c-gpio,i2c_gpio_sda=23,i2c_gpio_scl=24,i2c_gpio_delay_us=2

DPI Video Setup

dtoverlay=dpi18
overscan_left=0
overscan_right=0
overscan_top=0
overscan_bottom=0
framebuffer_width=854
framebuffer_height=480
enable_dpi_lcd=1
display_default_lcd=1
dpi_group=2
dpi_mode=87

dpi_output_format=458773
hdmi_timings=854 0 14 4 12 480 0 2 3 9 0 0 0 60 0 32000000 3

I2C
因为一开始就有投影，这个i2c 也有写在文件里

/etc/rc.local 文件修i2c

sudo i2cset -y 3 0x1b 0x0c 0x00 0x00 0x00 0x13 i
sudo i2cset -y 3 0x1b 0x0b 0x00 0x00 0x00 0x00 i

exit 0
修完之后重起树莓派
树莓派hdml的显示器就不能显示了，
dlp2000线如果接上了，就有投影了，

stm32 也可以驱动的，stm32 的ltdc 也就是dpi 接口，
但是有对芯片型号：stm32 并不是任何型号都支持ltdc
