springMVC项目访问静态资源时遇到的两个坑

1.存放静态资源文件夹放在了WEB_INF目录下，导致资源文件无法访问，最好放在项目根路径(WebRoot目录)下

2.引用资源文件路径采用了相对路径，由于资源文件路径随着controller访问路径变化而变化，导致资源文件引用不了，最好采用绝对路径引用资源文件，绝对路径示例：<%=basePath%>static/js/jquery.min.js, 

查看 jsp页面中是否设置basePath,设置如下：

<%
String path = request.getContextPath();
String basePath = request.getScheme()+"://"+request.getServerName()+":"+request.getServerPort()+path+"/";
%>  

static是我总的资源文件夹，下面有js、image、css等文件夹，里面分别存有对应资源文件。

我访问静态资源的方法是在springmvc-servlet.xml配置文件中加<mvc:resources mapping="/static/**" location="/static/" />
