原标题：Ai聘网之如何修改Spring Boot应用启动的嵌入式Tomcat的默认端口8080

Spring Boot是深受广大Java开发人员喜爱的框架，尤其是需要用Java开发微服务的那些开发人员。Sping Boot由Pivotal团队设计并完善，其设计目的是用来简化新Spring应用的初始搭建以及开发过程。该框架采取了约定优于配置的设计思路，从而使开发人员不再需要重复定义繁琐的样板化的配置。

我们知道Spring Boot启动应用时，会自动开启一个内置的Tomcat服务器，端口号默认为8080。见下图打印的日志输出。

如果开发人员想修改这个默认的端口号，该怎么办呢？本文Ai聘网介绍几种常用的修改Spring Boot自动启动的嵌入Tomcat服务器的默认端口号。

方法1：利用application.properties文件

在文件夹src/main/resources下面新建一个application.properties文件，

维护一行内容server.port=9091.

启动Spring Boot应用之后，观察到打印输出的日志里，Tomcat监听的端口号已经变成了9091。

方法2：利用setDefaultProperties

原理其实和方法1一样，只不过是用代码setDefaultProperties定义Tomcat的端口号，而不是application.properties文件。

SpringApplication app = new SpringApplication(Application.class);

app.setDefaultProperties(Collections.singletonMap("server.port", "8083"));

app.run(args);

方法3：利用ServerPortCustomizer

同样是用代码的方式修改端口号，原理和方法2不同。

@Component

public class ServerPortCustomizer

implements WebServerFactoryCustomizer {

@Override

public void customize(ConfigurableWebServerFactory factory) {

factory.setPort(8080);

}

}

方法4：使用参数--server.port=9999运行Spring Boot应用
