最近在研究selenium时，碰巧需要装configuration包，于是乎用pip在命令行安装

 结果报了：

2.问题解决

2.1 上网查浏览器

于是开始上浏览器搜，一开始全是升级setuptools，这里贴出来几句，可能你们的问题就解决了

pip install --upgrade pip
pip install --upgrade setuptools
pip install ez_setup

可我把这些执行了一遍，没解决问题，只是少了几行错(上面的图不是最开始的错)。

2.2 手动下载包解决

后面我想会不会是安装源的问题，仔细看，我报错的下面几行，提示源码错误(setup.py line 30)于是乎，找了国内流行的几个pip源(这里也给你们粘出来)

常见pip镜像源（国内源）
清华：https://pypi.tuna.tsinghua.edu.cn/simple
阿里云：http://mirrors.aliyun.com/pypi/simple/
中国科技大学 https://pypi.mirrors.ustc.edu.cn/simple/
华中理工大学：http://pypi.hustunique.com/
山东理工大学：http://pypi.sdutlinux.org/
豆瓣：http://pypi.douban.com/simple/

我准备对比一下这几个源下的包有什么不同，打开了清华源和豆瓣源，按ctrl+f 输入configuration，点进去会有各个时期的版本，大概是这样(豆瓣源)

 我下载了最新的0.4版本的。

之后便是解压，查看setup.py文件的第三十行代码,发现description不重要，于是直接给注释了(

#description = file(os.path.join(here, 'README.txt')).read()

)，注意修改后 ctrl+s 保存,后面又提示，我在方法上面加了一句，参考下图

description = ''

Traceback (most recent call last):
  File "setup.py", line 38, in <module>
    long_description=description,
NameError: name 'description' is not defined

 2.3 使用pip安装本地包

具体参考这篇文章

python使用pip安装本地包-Python之pip使用详解|附第三方库安装总结_weixin_37988176的博客-CSDN博客

我使用了2.离线安装

Step1:首先在https://pypi.org/下载需要的安装包

Step2:解压缩该文件。

Step3:命令行工具cd切换到所要安装的包的目录，找到setup.py文件，然后输入python setup.py install安装
