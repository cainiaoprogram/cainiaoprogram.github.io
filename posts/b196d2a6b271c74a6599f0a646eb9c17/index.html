使用PageMethod.startPage分页查询 返回总数对不上

在使用PageMethod.startPage方法进行分页查询操作时，如果对查询结果对象进行了转换，那么就会导致最后的数据总数结果对不上，导致前端显示的总数为当前页面的数量。

示例如下，总数仅仅是当前页的十条，但其实总数不止十条。

 问题代码：

​
 public PageInfo<ReceivedBO> listOrderPage(ReceivedQueryBO queryBO) {
        //设置从第几页开始，每页显示多少条
        PageMethod.startPage(queryBO.getPageNum(), queryBO.getPageSize());
        //构建查询sql和Example
        Example example = Example.builder(Received.class).where(queryBO.buildSql()).build();
        // 默认按照创建时间倒序排序
        example.setOrderByClause("create_time desc");
        List<Received> receiveds = receivedMapper.selectByExample(example);
        //将Received结果转换为ReceivedBO（对象发生变化）
        List<ReceivedBO> receivedBOList = ConverterUtils.convert(receiveds,ReceivedBO.class);
        return PageInfo.of(receivedBOList);
    }

​

解决

 public PageInfo<ReceivedBO> listOrderPage(ReceivedQueryBO queryBO) {
        //设置从第几页开始，每页显示多少条
        PageMethod.startPage(queryBO.getPageNum(), queryBO.getPageSize());
        //构建查询sql和Example
        Example example = Example.builder(Received.class).where(queryBO.buildSql()).build();
        // 默认按照创建时间倒序排序
        example.setOrderByClause("create_time desc");
        List<Received> receiveds = receivedMapper.selectByExample(example);
        //将Received结果转换为ReceivedBO（对象发生变化）
        List<ReceivedBO> receivedBOList = ConverterUtils.convert(receiveds,ReceivedBO.class);
        //问题解决关键
        PageInfo pageInfo = new PageInfo<>(receiveds);
        pageInfo.setList(receivedBOList);
        return pageInfo;
    }

​

这样，问题就成功解决啦！
