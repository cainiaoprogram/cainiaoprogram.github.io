小区选择、重选和轻松测量

1.小区选择

1.1、小区选择的目的

如果UE想要获取网络服务，就需要选择一个plmn下的小区驻留。在小区搜索过程中会搜到很多的小区，需要根据小区系统消息及终端属性确认当前小区是否适合驻留，小区电平和信号质量即是评价标准之一。

NR系统有多波束的概念，所以需要测量多个波束的测量值，只有超过门限的波束测量值才会进行最后的测量量计算。

对于多波束小区重选，包括从E-UTRA重选到NR，小区信道测量从SSB波束推导出来：

如果SIB2/SIB4没有配置nrofSS-BlocksToAverage参数，或者SIB2/SIB4没有配置absThreshSS-BlocksConsolidation参数，或者最高波束测量值小于等于absThreshSS-BlocksConsolidation。则认为小区测量值为最高波束测量值，每一个波束测量量在38.215定义。

否则，小区测量量为nrofSS-BlocksToAverage个高于门限absThreshSS-BlocksConsolidation的最高波束测量量功率的线性平均值。

1.2、小区选择的S准则

如果UE想在该小区驻留，需要满足如下条件：

Srxlev > 0 AND Squal > 0

Srxlev = Qrxlevmeas – (Qrxlevmin + Qrxlevminoffset )– Pcompensation - Qoffsettemp

Squal = Qqualmeas – (Qqualmin + Qqualminoffset) - Qoffsettemp

其中的参数分别解析

Srxlev

Cell selection RX level value (dB)

Squal

Cell selection quality value (dB)

Qoffsettemp

Offset temporarily applied to a cell as specified in TS 38.331 [3] (dB)

临时附加偏移，根据TS 38.331定义，connEstFailOffset在SIB1中定义，如果SIB1中没有该信元，则认为Qoffsettemp为无穷大。

如果SIB1配置了connEstFailureControl信元，且T300在一个小区连续超时connEstFailCount次，则对于connEstFailOffsetValidity指示的周期内，使用connEstFailOffset作为Qoffsettemp。T300定义为从UE发送RRCSetupRequest到收到RRCSetup或RRCReject时间。

 

connEstFailOffset

Parameter "Qoffsettemp" in TS 38.304 [20]. If the field is absent, the value of infinity shall be used for "Qoffsettemp".

  if the T300 has expired a consecutive connEstFailCount times on the same cell for which connEstFailureControl is included in SIB1:

  for a period as indicated by connEstFailOffsetValidity:

  use connEstFailOffset for the parameter Qoffsettemp for the concerned cell when performing cell selection and reselection according to TS 38.304 [20] and TS 36.304 [27];

Qrxlevmeas

Measured cell RX level value (RSRP)

Qqualmeas

Measured cell quality value (RSRQ)

Qrxlevmin

Minimum required RX level in the cell (dBm). If the UE supports SUL frequency for this cell, Qrxlevmin is obtained from q-RxLevMinSUL, if present, in SIB1, SIB2 and SIB4, additionally, if QrxlevminoffsetcellSUL is present in SIB3 and SIB4 for the concerned cell, this cell specific offset is added to the corresponding Qrxlevmin to achieve the required minimum RX level in the concerned cell;

else Qrxlevmin is obtained from q-RxLevMin in SIB1, SIB2 and SIB4, additionally, if Qrxlevminoffsetcell is present in SIB3 and SIB4 for the concerned cell, this cell specific offset is added to the corresponding Qrxlevmin to achieve the required minimum RX level in the concerned cell.

Qqualmin

Minimum required quality level in the cell (dB). Additionally, if Qqualminoffsetcell is signalled for the concerned cell, this cell specific offset is added to achieve the required minimum quality level in the concerned cell.

Qrxlevminoffset

Offset to the signalled Qrxlevmin taken into account in the Srxlev evaluation as a result of a periodic search for a higher priority PLMN while camped normally in a VPLMN, as specified in TS 23.122 [9].

Qqualminoffset

Offset to the signalled Qqualmin taken into account in the Squal evaluation as a result of a periodic search for a higher priority PLMN while camped normally in a VPLMN, as specified in TS 23.122 [9].

Pcompensation 

For FR1, if the UE supports the additionalPmax in the NR-NS-PmaxList, if present, in SIB1, SIB2 and SIB4:

max(PEMAX1 –PPowerClass, 0) – (min(PEMAX2, PPowerClass) – min(PEMAX1, PPowerClass)) (dB);

else:

max(PEMAX1 –PPowerClass, 0) (dB)

For FR2, Pcompensation is set to 0.

PEMAX1, PEMAX2

Maximum TX power level of a UE may use when transmitting on the uplink in the cell (dBm) defined as PEMAX in TS 38.101 [15]. If UE supports SUL frequency for this cell, PEMAX1 and PEMAX2 are obtained from the p-Max for SUL in SIB1 and NR-NS-PmaxList for SUL respectively in SIB1, SIB2 and SIB4 as specified in TS 38.331 [3], else PEMAX1 and PEMAX2 are obtained from the p-Max and NR-NS-PmaxList respectively in SIB1, SIB2 and SIB4 for normal UL as specified in TS 38.331 [3].

PPowerClass

Maximum RF output power of the UE (dBm) according to the UE power class as defined in TS 38.101-1 [15].

1.2.1、Qrxlevmeas

终端实际测量到的电平。PHY上报的参考信号接收功率，反映该小区的信号强度。

1.2.2、Qrxlevmin

驻留在该小区的最小接收电平，是SIB消息中配置的参数。如果是计算当前服务小区的Srxlev，则查看SIB1消息的配置。如果UE支持SUL频率且q-RxLevMinSUL存在时，优先使用q-RxLevMinSUL，否则使用q-RxLevMin。小区配置的值越大，使得该小区更难符合S准则，则UE选择驻留该小区的难度越大。

该参数在SIB2\SIB4\SIB5中都存在，分别是用在同频小区、异频小区、异系统小区的计算中。

1.2.3、q-RxLevMinOffset

正常驻留在VPLMN中时定期搜索更高优先级PLMN，需要在Srxlev评估中考虑了信号Qrxlevmin的偏移，为的是降低找到更高优先级PLMN的门槛。如果是计算当前服务小区的Srxlev，则查看SIB1消息的配置，如上图。该参数在SIB3\SIB4\SIB5中都存在，分别用在同频小区、异频小区、异系统小区的计算中。

1.2.4、Pcompensation

这个是功率惩罚值。下图是一个示意图，假如小区配置的最大发射功率是Pmax，但是终端允许的最大发射功率是PPowerClass，且PPowerClass小于Pmax，就需要考虑这个惩罚因子max(PEMAX1–PPowerClass, 0) (dB)。当然现在NR中可能存在additionalPmax，则新的计算方法如下

max(PEMAX1 –PPowerClass, 0) – (min(PEMAX2, PPowerClass) – min(PEMAX1, PPowerClass)) (dB);

 

PEMAX1可以分别对应SIB1、SIB3、SIB5里面的P-max，PEMAX2对应additionalPmax。

SIB1、SIB3、SIB5中的P-max分别针对服务小区，同频邻区以及异频邻区，理论上这三个值可以设计各不相同，但是设置是必选的。

如果支持上行补充频点，则PEMAX1和PEMAX2选择补充频点对应的P-max和additionalPmax.

针对多次T300超时的小区引入的惩罚因子，即UE多次在该小区上发送RRCSetupRequest，但是未收到RRCSetup消息。该参数在SIB1中指定：

 

如果T300定时器超时次数超过了connEstFailCount，则会启动周期为nnEstFailOffsetValidity的定时器，在定时器running期间，计算该小区的服务质量时，需要引入临时偏置connEstFailOffset.

1.2.6、Qqualmin

驻留在该小区的最小接收质量水平，是SIB消息中配置的参数。如果是计算当前服务小区的Srxlev，则查看SIB1消息的配置。该参数在SIB2\SIB4\SIB5中都存在，分别是用在同频小区、异频小区、异系统小区的计算中。

1.2.7、Qqualminoffset

正常驻留在VPLMN中时定期搜索更高优先级PLMN，需要在Squal评估中考虑了信号Qqualmin的偏移，为的是降低找到更高优先级PLMN的门槛。如果是计算当前服务小区的Squal，则查看SIB1消息的配置。该参数在SIB3\SIB4\SIB5中都存在，分别用在同频小区、异频小区、异系统小区的计算中。

2.小区重选

2.1、重选优先级

不同NR频点或者异系统频点优先级可能来自于SIB、RRCRelease或者异系统重选时来自于异系统。如果SIB中异频没有配置重选优先级，则不进行小区重选测量。

如果专用信令配置了频点优先级，则UE忽略所有来自于SIB的优先级。

如果UE处于camped on any cell状态（未注册成功，只有acceptable cell），则UE只使用当前小区SIB配置的优先级，保留专用信令和RRCRelease 信令deprioritisation信元配置的优先级。

当UE处于camped normally状态（注册成功，正常驻留），当前频点没有优先级，异频配置了优先级时，UE认为当前频点为最低优先级。

在RRC状态改变、T320超时、NAS发起PLMN选择时，要删除所有的频率优先级。
不同RAT的频率优先级不可以设置为相同的值。

2.1.1、deprioritisationReq作用

当UE收到的RRCRelease消息携带deprioritisationReq信元，UE认为当前频点和之前存储的通过RRCRelease信令中deprioritisationReq信元收到的频点或者整个NR系统的频点优先级最低，持续时间deprioritisationTimer为T325。

  deprioritisationReq                 SEQUENCE {

        deprioritisationType                ENUMERATED {frequency, nr},

        deprioritisationTimer               ENUMERATED {min5, min10, min15, min30}

2.1.2、专用信令提供优先级

RRCRelease消息中可能携带频点优先级CellReselectionPriorities，在T320超时前，RRC_IDLE状态的UE应该使用专用信令中配置的频点优先级。以下情况下UE删除专用信令提供的优先级：

- UE进入另一个RRC状态。

- 专用优先级有效时间定时器T320超时。

- UE收到RRCRelease消息不带cellReselectionPriorities信元

 - NAS指示PLMN选择或者SNPN选择

2.1.3、CellReselectionSubPriority

SIB2、SIB4、SIB5和RRCRelease消息的频点优先级信息新增了参数CellReselectionSubPriority，CellReselectionSubPriority加上cellReselectionPriority用于计算频点的绝对优先级。

CellReselectionSubPriority

The IE CellReselectionSubPriority indicates a fractional value to be added to the value of cellReselectionPriority to obtain the absolute priority of the concerned carrier frequency for E-UTRA and NR. Value oDot2 corresponds to 0.2, value oDot4 corresponds to 0.4 and so on.

CellReselectionSubPriority information element

CellReselectionSubPriority ::=          ENUMERATED {oDot2, oDot4, oDot6, oDot8}

2.1.4、边缘通信（sidelink communication）

如果配置UE同时进行NR边缘通信和V2X边缘通信，UE认为同时提供NR边缘通信和V2X边缘通信的频点为最高优先级。如果配置UE只进行NR边缘通信不进行V2X边缘通信，UE认为提供NR边缘通信的频点为最高优先级。如果配置UE不进行NR边缘通信只进行V2X边缘通信，UE认为提供V2X边缘通信的频点为最高优先级。

The cell-ranking criterion Rs for serving cell and Rn for neighbouring cells is defined by:

Rs = Qmeas,s +Qhyst - Qoffsettemp

Rn = Qmeas,n -Qoffset - Qoffsettemp

where:

Qmeas

RSRP measurement quantity used in cell reselections.

Qoffset

For intra-frequency: Equals to Qoffsets,n, if Qoffsets,n is valid, otherwise this equals to zero.

For inter-frequency: Equals to Qoffsets,n plus Qoffsetfrequency, if Qoffsets,n is valid, otherwise this equals to Qoffsetfrequency.

Qoffsettemp

Offset temporarily applied to a cell as specified in TS 38.331 [3].

2.3.1 Qoffset

服务小区和邻区之间的偏移量，用于控制小区重选的难度。给某个邻区配置的偏移量越大，则越难重选到该小区。该小区有正负值。

在相同信号条件下，计算得到服务小区和邻区的R值相同，但是这两个R值对于UE选择服务小区和邻区而言，合适的程度可能不同。比如，小区1的最低允许驻留的门限为-100dBm，小区2的最小允许驻留门限为-98dBm，UE同时测量到A和B的RSRP值均为-98dB。假设其他条件相同，则计算得到的R值相同，但显然小区1更适合驻留。因此小区重选计算邻区R值，需要引入一个offset，来保证不同小区的R值更公平。

2.3.5移动场景UE测量

状态检测：

如果SIB中配置了TCRmax, NCR_H, NCR_M and TCRmaxHyst参数，则根据参数确定UE移动状态。

Normal-mobility state：周期TCRmax内小区重选次数少于NCR_M

Medium-mobility state：周期TCRmax内小区重选次数大于等于NCR_M并且少于NCR_H。

High-mobility state：周期TCRmax内小区重选次数大于NCR_H。

计算重选次数时，不考虑连续重选，连续重选指根据移动状态检测重选一个小区后又重选到该小区。

移动状态改变：

 - 如果满足高速状态检测，则进入High-mobility state

 - 如果满足中速状态检测，则进入Medium-mobility state

 - 在周期TCRmaxHyst内不满足高速和中速状态检测，则进入Normal-mobility state

移动性缩放准则如下：

如果处于Normal-mobility state，则不进行参数缩放

如果处于High-mobility state

  - 如果SIB中配置了q-HystSF，则将Qhyst加上sf-High的q-HystSF。

  - 如果配置了t-ReselectionNR-SF，则将TreselectionEUTRA或TreselectionNR乘以sf-High的t-ReselectionNR-SF

如果处于Medium-mobility state

  - 如果SIB中配置了q-HystSF，则将Qhyst加上sf-Medium的q-HystSF。

  - 如果配置了t-ReselectionNR-SF，则将TreselectionEUTRA或TreselectionNR乘以sf-Medium的t-ReselectionNR-SF

对于高速或中速状态，乘以t-ReselectionNR-SF后得到的TreselectionRAT向上取整。

3.轻松测量（relaxedMeasurement）

相比LTE IDLE态测量增加了轻松测量准则，在满足轻松测量判断准则情况下，加长测量周期，以降低功耗。

有三种轻松测量准则，一种是判决用户处于低速时，一种是判决用户不在小区边缘，还有一种联合判断。

 

3.1判断准则

①、轻松测量低速移动用户判断准则：

(SrxlevRef – Srxlev) < S_SearchDeltaP 持续时间T_SearchDeltaP时满足低速UE轻松测量准则。

其中Srxlev为服务小区当前Srxlev值。

SrxlevRef为服务小区Srxlev参考值，计算方法为：

如果选择或重选到一个新小区

或者 (Srxlev - SrxlevRef) > 0

或者T_SearchDeltaP 时间内不满足轻松测量准则

则UE设置SrxlevRef为Srxlev

在重选到新小区时，设置参考值SrxlevRef = Srxlev，如果当前值Srxlev变大则设置参考值SrxlevRef = Srxlev，如果当前值变小，则参考值不变。如果T_SearchDeltaP 时间内不满足轻松测量准则，则设置参考值SrxlevRef = Srxlev。可以理解为参考值SrxlevRef为过去一段时间内服务小区Srxlev的最大值，如果当前值与参考值的差值小于S_SearchDeltaP，则认为UE收到的RSRP处于稳定状态，间接判断出UE处于低速状态，可以进行轻松测量。

②、轻松测量非小区边缘用户判断准则

-     Srxlev > SSearchThresholdP, and,

-     Squal > SSearchThresholdQ, if SSearchThresholdQ is configured,

3.2.1、低速场景

低速场景判断

1、配置了lowMobilityEvaluation参数，没有配置cellEdgeEvaluation参数，

2、UE在重选到新小区进行正常同频、异频或异系统测量超过T_SearchDeltaP时间

3、在周期T_SearchDeltaP内满足上节介绍的低速移动用户轻松测量准则。

低速场景测量准则：

1、 UE根据TS 38.133 4.2.2.9节介绍的方法进行同频小区轻松测量。

2、如果服务小区满足Srxlev > SnonIntraSearchP且 Squal > SnonIntraSearchQ，异频/异系统高优先级小区配置了highPriorityMeasRelax时每小时测量一次（每小时测量一个频点），没有配置highPriorityMeasRelax时，根据TS 38.133进行正常测量。

3、如果不满足Srxlev > SnonIntraSearchP and Squal > SnonIntraSearchQ，根据TS 38.133中4.2.2.10、4.2.2.11对异频和异系统频点进行轻松测量。

3.2.2、非小区边缘场景

非小区边缘场景判断

1、配置了cellEdgeEvaluation参数，没有配置lowMobilityEvaluation参数

2、并且满足上节非小区边缘场景的轻松测量准则。

非小区边缘场景测量准则：

 1、UE根据TS 38.133中4.2.2.9节对同频小区进行轻松测量。

 2、如果服务小区满足Srxlev ≤ SnonIntraSearchP或Squal ≤ SnonIntraSearchQ，UE根据TS 38.133中4.2.2.10、4.2.2.11对异频或异系统小区进行轻松测量。对于Srxlev > SnonIntraSearchP and Squal > SnonIntraSearchQ时，高优先级频点，测量时间与普通场景一致。

 

3.2.3、联合场景

联合场景判断

1、lowMobilityEvaluation和cellEdgeEvaluation参数均配置了：

2、UE在重选到新小区后进行正常同频、异频或异系统测量超过TSearchDeltaP

3、周期TSearchDeltaP内满足上节介绍的低速场景轻松测量准则

4、满足上节介绍的非小区边缘场景轻松测量准则

联合场景测量准则：

1、如果判断满足低速、非小区边缘轻松测量准则，则一小时进行一次同频、异频、异系统测量。

2、如果只满足低速或者非小区边缘中一个轻松测量准则，且没有配置combineRelaxedMeasCondition参数，UE根据TS 38.133 4.2.2.9, 4.2.2.10和4.2.2.11节轻松测量准则测量同优先级或低优先级的同频、异频、异系统频点。如果服务小区满足Srxlev ≤ SnonIntraSearchP or Squal ≤ SnonIntraSearchQ，根据TS 38.133 4.2.2.10和 4.2.2.11 节轻松测量准则进行高优先级异频或异系统频点测量。

3、如果只满足低速或者非小区边缘中一个轻松测量准则，且combineRelaxedMeasCondition参数配置了，则不进行轻松测量。
