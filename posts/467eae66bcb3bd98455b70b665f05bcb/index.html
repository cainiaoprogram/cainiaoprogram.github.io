电脑开机显示profsvc服务器登陆失败,User Profile Service服务登录失败 无法加载用户配置文件...

ProfSvc 、User Profile Service 、 Windows 10

前不久电脑遇到开机的时候弹提示：

ProfSvc服务登录失败。无法加载用户配置文件

点击确认后又可以登录就不以为意，直到今天我再次打开电脑，又是弹出这个提示，然后自动注销用户，重新进入用户登录界面，输入用户的PIN后确认却无法进入系统，弹出：

User Profile Service服务登录失败。无法加载用户配置文件

反复数次都是这样，重启还是这样，在网上找的办法都是删除登录的用户所在的C:/Users/*username/*目录，然后删除注册表里对应用户的条目。可是我都无法进入系统，我怎么删除？？？

在翻找了很多信息后瞎弄来弄去重启还是不信，于是我鬼使神差的点击了用户后使用密码登录(之前是使用用户PIN码登录一直报错)，然后竟然奇迹般进去了，不过注销用户后使用PIN还是老毛病，又在网上找了很多解决方案，都是有些零零散散的信息，难以去实验。

好像在知乎看到一条说什么C:/Users/Default/目录有什么问题的，还有说登录用户目录下的NTUSER.DAT 文件有问题，需要从别的地方重新弄一个什么的。于是我去看了我的C:/Users/目录，发现没有Default目录,恍然大悟，才想起前不久整理磁盘空间，好像删除了一些以前创的一些本地账户没用了，把Default目录也给删除了，然后我手动创建了Default目录，按照网上的说法以为系统会自动在这个目录下生成特定的文件，但是并没有，于是我从另一个电脑复制了它的Default到我这个电脑上，完事儿注销用户，再次登录竟然没有任何告警了，重启后也没有任何告警了。hhh

猜测Default应该是系统在没有用户登录的时候的默认桌面和系统设置文件吧，如果删除了，系统将无法预配置系统？？？仅猜测一下，欢迎指正！

当然，如果使用用户密码也无法进入系统，在安全模式下复制或新建一个Default目录然后重启也可以尝试。
