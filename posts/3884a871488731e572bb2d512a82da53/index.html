添加服务器共享文件,如何在 VMM 中添加 Windows 文件服务器共享

适用于： System Center 2012 SP1 - Virtual Machine Manager,System Center 2012 R2 Virtual Machine Manager,System Center 2012 - Virtual Machine Manager

使用以下过程来添加远程基于 Windows 的文件服务器中的存储设备作为 Virtual Machine Manager (VMM)。

帐户要求 要完成此过程，您必须是管理员用户角色的成员或委派的管理员用户角色的成员。

在开始此过程之前，您可以创建运行方式帐户之前或同时使用添加存储设备向导来发现存储。 运行方式帐户必须在文件服务器上具有管理员权限。 VMM 使用此帐户以执行管理操作，例如，若要创建共享并修改服务器上的共享权限。 有关运行方式帐户的详细信息，请参阅 如何在 VMM 中创建运行方式帐户。

添加文件服务器

当您将添加文件服务器上， VMM 自动发现当前未显示在服务器的所有共享。

若要添加的文件服务器

打开 构造 工作区。

在 构造 窗格中，单击 存储。

在 主页 选项卡上，单击 添加资源, ，然后单击 存储设备 以启动添加存储设备向导。

在 选择提供程序类型 页上，选择 将基于 Windows 的文件服务器添加为被管理的存储设备 管理中的单引号或群集文件服务器 VMM 控制台。

在 指定发现作用域 页上，执行以下操作︰

在 提供程序 IP 地址或 FQDN 框中，指定的地址或文件服务器的名称。

如果文件服务器位于不受信任的所在的域中的虚拟机主机，选择一个域 这台计算机处于不受信任的 Active Directory 域。 虚拟机主机使用文件服务器存储。

接下来为 运行方式帐户 框中，单击 浏览, ，并选择可访问存储提供程序的运行方式帐户。 如果尚无帐户，请执行以下操作。 单击 浏览, ，然后在 选择运行方式帐户 对话框中，单击 创建运行方式帐户。

在 收集信息 页上， VMM 会自动尝试发现并导入有关共享文件服务器上的信息。 如果发现过程成功，将显示有关文件服务器的信息。 在该过程完成后，单击 下一步。

在 选择存储设备 页上，选择所需的文件共享旁边的复选框 VMM 来管理。

在 摘要 页上，确认这些设置，然后单击 完成。

作业 对话框随即出现。 确保作业的状态为 已完成, ，然后关闭该对话框。

若要验证的新发现的存储信息，在 构造 工作区中，请在 主页 选项卡上，单击 构造资源。 在 构造 窗格中，展开 存储 节点。

之后添加并且发现文件服务器存储时，可以将 SMB 3.0 文件共享分配给主机和主机群集中。 有关详细信息，请参阅 如何分配给 HYPER-V 主机和群集在 VMM 中的 SMB 3.0 文件共享。
