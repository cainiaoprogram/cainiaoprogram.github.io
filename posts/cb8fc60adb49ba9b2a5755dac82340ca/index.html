将团队项目门户移动到运行 SharePoint 产品和技术的其他服务器上

08/18/2008

本文内容

更新：2007 年 11 月

如果当前服务器和新服务器正在运行 Windows SharePoint Services 3.0 或 Microsoft Office SharePoint Server 2007，则可将团队项目门户移动到其他服务器上。例如，可将项目门户从 Visual Studio Team System Team Foundation Server 的应用层服务器移动到承载其他网站的服务器上。

若要将项目门户移动到其他服务器上，您先要迁移每个团队门户。然后，要在新服务器上安装用于 SharePoint 产品和技术的 Team Foundation Server 扩展。最后，您将更新 Team Foundation Server，以便在新服务器上查找和创建团队站点。

移动团队站点

如果项目门户当前在运行 Windows SharePoint Services 2.0 的服务器上，则在移动项目门户之前必须将该服务器升级到 Windows SharePoint Services 3.0 或 Microsoft Office SharePoint Server 2007。有关更多信息，请参见升级用于 Team Foundation Server 的 SharePoint 产品和技术。

若要将项目门户移动到运行 Windows SharePoint Services 3.0 的服务器上，请参见 Microsoft 网站上的 Migrate content or sites after upgrade (Windows SharePoint Services)(升级后迁移内容或站点 (Windows SharePoint Services))。若要将项目门户移动到运行 Microsoft Office SharePoint Server 2007 的服务器上，请参见 Microsoft 网站上的 Migrate content or sites after upgrade (Office SharePoint Server)(升级后迁移内容或站点 (Office SharePoint Server))。

重要说明：

如果要将项目门户移动到具有现有站点的服务器上，则必须遵循迁移主题中介绍的导入/导出方法。

安装扩展

若要在新服务器上安装用于 SharePoint 产品和技术的 Team Foundation Server 扩展，必须访问安装介质、打开 WssExt 文件夹并运行该文件夹中的安装程序。

更新 Team Foundation Server

安装了扩展后，必须使用 TFSAdminUtil.exe 来指示 Team Foundation Server 使用已将项目门户移动到的服务器。

TfsAdminUtil configureconnections /SharepointURI:http://SharepointServer:Port/sites /SharepointAdminUri:http://SharepointServer:AdminPort /SharepointUnc:\\SharepointServer\sites

下表描述了要在命令提示符处使用的值：

SharepointServer

在其上部署 SharePoint 产品和技术的服务器的名称。

Port

SharePoint 产品和技术用于用户服务的端口。此值通常为 80。

AdminPort

SharePoint 产品和技术用于管理服务的端口。

说明：

即使您是使用管理凭据登录的，也必须使用提升的权限打开命令提示窗口，才能在正在运行 Windows Server 2008 的服务器上执行此功能。若要使用提升的权限打开命令提示窗口，请单击“开始”，右击“命令提示符”，然后单击“以管理员身份运行”。有关更多信息，请参见 Microsoft 网站。
